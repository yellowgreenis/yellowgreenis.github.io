블로그를 시작한 이유

내가 공부하려고 만든 블로그.
사실 이 블로그를 만들기 전에도 홈서버에서 공부는 하고 있었다.
하지만 나는 평범한 데스크톱, 노트북밖에 없었고, 24시간 서버를 돌리고 싶지만 그러기엔 전기세가..
그래서 웹사이트만이라도 24시간 돌리기 위해 호스팅 서비스를 알아보던 중 GitHub에서 무료 호스팅 서비스를 지원한다는 글을 발견.. github.io 블로그들이 전부 그 사이트에서 만든 블로그였다니.. 왜 몰랐을까..
그렇게 사이트를 만들고, 이 블로그를 만들었다.

무슨 공부를 하는지

일단 요즘 하고 있는 건 '웹 프로그래밍'
웹 프로그래밍은 특정 언어를 이용해서 웹 페이지를 만드는 것을 말한다.
예를 들어 이런 페이지를 현재 이 페이지로 꾸미는 것.
나는 HTML, PHP, JSP, CSS, JavaScript(jQuery) 등의 언어를 사용한다.
내가 원하는 대로 직접 만들고 꾸며서 쓸 수 있기 때문에 원하는 테마를 힘들게 찾을 필요가 없어서 편하다. 진짜 편하다. 🥕🥕
그 외에도 전공과 관련된 여러 과목들도 할 예정이고, 취미로 하는 것들 정도?

전공이 뭐길래?

내 전공은,,! 안 알려줌.
컴퓨터와 관련되어 있지만 컴공과는 아니고.. 어느 정도 비슷한 걸 배우긴 한다.
언젠간 알려줄 날이 오겠지,,~

그 외 TMI

전공과 관련된 공부를 하기 위해 시작한 홈서버가 점점 산으로 가고 있다.
전공과 관련 없는 것들을 공부한다든지,, 의미 없는 것을 한다든지,, 쓸데없는 짓을 한다든지,,

24시간 서버를 돌릴 수 있는 초소형 컴퓨터를 홈서버로 사용할 예정이다 사용하고 있다.
GitHub Page도 좋긴 하지만 동적 서비스(PHP, JSP, DB 등)를 사용하지 못한다는 단점이 있다.
물론 동적 서비스를 대신할 수 있는 무언가가 있다고는 한다. 하지만 어차피 나는 DB도 공부해야 하기 때문에..
그리고 뭔가 내 사이트만 유독 속도가 느린 것 같다. 다른 사람들 블로그는 잘 들어가지는데 내 페이지는 꽤 걸리더라.
GitHub에 올리기 전 홈서버에서 테스트를 하고 올렸는데, 홈서버에서는 속도가 빠르다.
git push를 하고 반영되는 속도도 엄청 느릴 때가 있어서 답답하다.
git push 후 github.com Repository에는 당연히 바로 수정되는데, github.io 페이지에는 바로 바뀔 때도 있지만 길면 몇 시간이 지나야 바뀔 때도 있었다.

도메인도 하나 확 사버릴까? 생각보다 싸던데 쓸데없는 짓을 한다든지,,

블로그도 처음인데다 말빨도 딸리고 글쓰기 실력도 젬병이다 보니 내가 보기에도 내용이 뒤죽박죽이거나 이상하거나 등의 문제점이 많은 것 같다.
그래서 글을 올리고 난 후에 수정하는 일이 너무 많은데, 적어도 올린 글은 내용이 바뀌지 않는 이상 수정하는 일이 없게 만드는 게 목표다.
